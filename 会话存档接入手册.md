# 企业微信「会话内容存档」接入手册

> 官方文档：https://developer.work.weixin.qq.com/document/path/91360
> ⚠️ 仅在 **Linux 服务器**运行（需原生 SDK 的 `.so`）。本地 Windows 开发机不支持。
> 作用：把真实群聊消息拉进系统 → 客服效能/AI日报/情绪/查件触发 全部用真实数据。

---

## 0. 密钥说明（重要，别搞反）

会话存档用一对 RSA 密钥：
- **公钥 PUBLIC KEY** → 上传到企微后台（你已提供，确认已传后台并记下「版本号 publickey_ver」）
- **私钥 PRIVATE KEY** → **留在服务器**，本系统用它解密。**这才是代码需要的那把。**

> 公钥不能解密，只有私钥能。你之前发的是公钥；解密要配套的**私钥**文件（`-----BEGIN PRIVATE KEY-----` 或 `RSA PRIVATE KEY` 开头）。
> 私钥很敏感——**别贴进聊天/不要提交 Git**，直接放服务器文件。

---

## 1. 企微后台准备

1. 「管理后台」→「安全与管理」→「会话内容存档」：开通（需企业认证、购买相应席位）。
2. 设置消息加密公钥（把你那把**公钥**贴进去），记下版本号。
3. 获取「**会话存档 Secret**」（与普通应用 Secret 不同）。→ 已配到 `.env` 的 `WX_ARCHIVE_SECRET`。
4. 配置可信 IP = 服务器公网 IP。
5. 合规：按企微要求完成对员工/客户的会话存档告知。

## 2. 服务器放置 SDK 与私钥

```bash
# 下载官方 SDK（C版），取其中的 .so 放到项目 sdk 目录
mkdir -p /opt/wxwork_analytics/sdk /opt/wxwork_analytics/secrets
# 把 libWeWorkFinanceSdk_C.so 放到 sdk/
cp libWeWorkFinanceSdk_C.so /opt/wxwork_analytics/sdk/
# 私钥写入文件（仅本机可读）
vi /opt/wxwork_analytics/secrets/wx_archive_private_key.pem   # 粘贴 PRIVATE KEY
chmod 600 /opt/wxwork_analytics/secrets/wx_archive_private_key.pem
```
> Docker 部署时把这两个目录挂载进 backend 容器，并确保容器内能 `dlopen` 该 .so（基础镜像装好依赖）。

## 3. 配置 .env（服务器）

```
WX_ARCHIVE_CORPID=ww你的企业ID         # 留空则用 WX_CORP_ID
WX_ARCHIVE_SECRET=（会话存档Secret，已填）
WX_ARCHIVE_SDK_PATH=/opt/wxwork_analytics/sdk/libWeWorkFinanceSdk_C.so
WX_ARCHIVE_PRIVATE_KEY_PATH=/opt/wxwork_analytics/secrets/wx_archive_private_key.pem
```

## 4. 验证与运行

- 状态：`GET /api/archive/status` → `{"available": true}` 表示配置齐全
- 手动同步：`POST /api/archive/sync` → 后台拉取并写入消息（游标 seq 自动保存，增量同步）
- 定时：用 1Panel 计划任务或 cron 每 N 分钟调用一次同步（或后续加内置定时器）

同步后，群消息进入 `wx_message` 表，所有分析自动变为真实数据。

## 5. 自动查件闭环（同步打通后即可加）

客户在群里发"我的货到哪了" → 命中查件意图 → 用 群↔客户 映射拿到 user_number → 调 NextSLS 查件 → 轨迹推给客服一键发出。
（这步等会话存档跑通、确认真实消息进库后再开，避免空跑。）
